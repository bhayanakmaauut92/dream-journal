### 3. RIVERSIDE → HYPERFRAMES WORKFLOW
**Agentic editing pipeline (MVP in 48 hours)**:
1. **Raw capture**: Record via Riverside → auto-save to `/recordings/`
2. **Transcription**: `npx hyperframes transcribe input.mp4 --output transcript.json`
3. **Viral detection**: Run keyword/energy scoring script (see below)
4. **Clip cutting**: FFmpeg cuts at detected timestamps (`ffmpeg -ss 00:01:23 -t 00:00:25 -i input.mp4 -c copy clip.mp4`)
5. **Composition**: Hyperframes HTML template with dynamic data injection
6. **Render**: `npx hyperframes render composition.html -o output.mp4 --quality high --fps 15`
7. **Delivery**: Gofile.io upload → Telegram link

**Automatable NOW** (Steps 2, 3, 4, 5, 6, 7):
- Transcription (Whisper via Hyperframes)
- Viral detection (keyword matching + audio energy threshold)
- FFmpeg cutting (timestamp-based)
- Hyperframes composition (template with placeholders)
- Rendering (CLI command)
- Gofile upload (existing script)

**Manual work** (Week 1 only):
- Approving clip selections (reduce to 5 seconds/clip)
- Minor text tweaks in composition

**Simple MVP pipeline**:
```bash
# 1. Get latest recording
ls -t /recordings/ | head -1 > latest.txt
```

### 10. ORGANIC CONVERSATION AUTOMATION
**Safe automation quotas** (per account, warm-up phase):
- Likes: 50/hour max (spread evenly, varied content)
- Follows: 20/day max (with 2-day cooldown per follow)
- Replies: 15/hour max (only to posts <6 hours old, must read first 2 lines)
- DMs: 5/day max (only to new followers, welcome message template)

**Human-in-the-loop architecture**:
- **Automated**: Likes, follows, welcome DMs, scheduled posts
- **Manual**: All replies requiring context, comment threads, controversy engagement
- **Hybrid**: AI suggests reply templates → human edits/approves before sending

**Safe tools & methods**:
- **X**: Use Twitter API v2 with elevated access (apply for academic/research) + custom rate limiting
- **IG/YT**: Official APIs with built-in rate limits (use cautiously)
- **Alternative**: Mobile device farming (3-5 old Androids) running scripts via ADB (appears as genuine mobile usage)
- **Warming up**: Start at 10% of quotas for Week 1, increase 20% every 3 days

**Reply templates that don't sound bot-like** (customize per post):
- "Interesting take! How did this work for [specific business type]?"
- "Saw this help my friend's [similar business] — curious about your experience"
- "Question: Does this integrate with [common tool they likely use]?"
- "Thanks for sharing! What's the biggest surprise you found?"

**What NOT to automate**:
- Replies to negative/complaint posts
- Engagement with political/controversial topics
- Following inactive accounts (>90 days no posts)
- Copy-paste identical comments
- Actions triggering CAPTCHAs (pause and verify human)

**Anti-bot signal avoidance**:
- Vary timing between actions (randomized 30-90s delays)
- Complete profiles (bio, photo, banner, website)
- Post organic content daily (algorithm favors accounts that create content)
- Mix engagement types (don't just like — also retweet, quote, reply)
- Use residential proxies if scaling (avoid datacenter IPs)
