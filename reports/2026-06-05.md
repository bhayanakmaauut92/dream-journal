### 1B — Sync Method Analysis and Recommendation

**Options for syncing 3 video files in post-production:**

| Method | Accuracy | Cost | User Skill Needed | How It Works |
|--------|----------|------|-------------------|--------------|
| **Audio Clap (Recommended)** | ±1-2 frames | Free | None | Everyone claps on "3" at start. Editor aligns the audio waveform spikes. |
| Time-of-Day Timecode | ±2-5 frames | Free | Low | Each host shows time.navy.mil on phone for 5s at start. Editor matches timecodes. |
| PluralEyes Auto-Sync | ±1 frame | $199 one-time | Low | Software auto-analyzes waveforms and synchronizes all clips in one click. |
| Tentacle Sync E Hardware | Frame-accurate | €200/unit (need 3) | Medium | Generates LTC timecode signal that OBS records on a separate audio track. |

**RECOMMENDATION: Audio Clap + PluralEyes**

For the first few episodes, use the clap method. It costs nothing, requires no extra hardware, and works within 1-2 frames accuracy — which is imperceptible to viewers at 30fps.

**The clap protocol (teach your hosts):**
1. Ryan counts down: "Clap on 3... 3... 2... 1... CLAP"
2. Everyone claps once, loudly, directly toward their microphone
3. All three claps land within the same ~200ms window
4. In your editor (DaVinci Resolve or Premiere), zoom into the audio waveform — the clap creates a massive spike visible across all three tracks. Align them.

**When to upgrade to PluralEyes:** If you're editing 3+ episodes per week and the manual alignment (15-20 seconds per episode) becomes annoying. PluralEyes is $199 one-time and auto-syncs in one click. Worth it for volume editing.

**Hardware sync is overkill** for this setup. Tentacle Sync is for multi-camera live events with 4+ cameras. You have 3 webcams at 30fps — the clap method gives you exactly the same accuracy for free.

### 1C — Babio's Post-Production Workflow (DaVinci Resolve)

**Step-by-step guide for stitching 3-host video in DaVinci Resolve:**

**Phase 1: Import**
1. Create a new project: "AIMN Episode [Date]"
2. Right-click in Media Pool → Import Media → Select all 3 host files
3. Right-click each file → Clip Attributes → Confirm all are 1920x1080, 30fps, 48kHz audio

**Phase 2: Align by Waveform (the clap sync)**
1. Create a new timeline (File → New Timeline → 1920x1080, 30fps)
2. Drag all 3 clips into the timeline, stacking them on different video tracks (V1, V2, V3)
3. Zoom in on the waveform at the beginning of the timeline
4. Find the clap spike on Track 1 (Ryan's video)
5. Slide Tracks 2 and 3 so their clap spikes align perfectly with Track 1's spike
6. Fine-tune at the sample level: zoom into the spike and adjust frames until they're exactly aligned

**Phase 3: Create Multicam Clip**
1. Select all 3 aligned clips (hold Shift + click each)
2. Right-click → "New Multicam Clip Using Selected Clips..."
3. Name it "AIMN Episode [Date] - Multicam"
4. Check "Use audio from none — manually choose each cut" (you'll switch angles manually)
5. Delete the original 3 clips from the timeline
6. Drag the multicam clip onto the timeline

**Phase 4: Edit with Multicam Viewer**
1. Go to Workspace → Video Clean Feed (full preview of your output)
2. Find the Multicam Viewer: Workspace → Show Multicam Viewer (if available in your Resolve version)
   - OR: Double-click the multicam clip to open it in the Source Viewer
   - OR: In the Edit page, enable the "Angle Viewer" overlay
3. Play through the timeline. When you want to switch to a different host's camera, click their angle thumbnail or press 1/2/3 on the keyboard
4. The video switches instantly, creating cuts at each decision point

**Alternative (no multicam viewer):** Simply keep all 3 clips on separate tracks, and use the Razor tool (B key) to cut away portions you don't want. Mute (Shift + M) the tracks you want to hide.

**Phase 5: Add Overlays & Polish**
- Use your AIMN broadcast overlay template for header bar/logo/ticker over the full-episode export
- Add lower-thirds (host name/title) when a new person speaks
- Trim dead air / pauses using the Razor tool or silence detection

**Phase 6: Export**
1. Deliver page → Custom Export
2. Format: MP4
3. Codec: H.264
4. Resolution: 1920x1080
5. Frame Rate: 30
6. Quality: Restrict to 20000 Kbps
7. Audio: AAC, 320 Kbps, 48 kHz
8. Add to Render Queue → Render All

**Phase 7: Upload to Riverside.fm** (if you're using it only for distribution)
- Create a new episode in Riverside.fm
- Upload the exported MP4
- Riverside handles hosting, player, analytics, and download links

**Expected editing time per episode:** 45-90 minutes for a 45-minute episode once you're practiced. The clap alignment takes 30-60 seconds. The multicam cutting takes the longest.

### 1E — Quality Comparison: Riverside.fm vs OBS Local Recording

| Aspect | Riverside.fm | OBS Local Recording | Winner |
|--------|-------------|-------------------|--------|
| **Video Resolution** | Up to 4K (compressed, variable) | Native 1080p (no intermediate compression) | OBS |
| **Video Bitrate** | ~8 Mbps effective (variable, depends on connection) | 18 Mbps CBR (you control this exactly) | OBS — 2.25x more data per frame |
| **Audio Quality** | 48 kHz AAC (cloud-processed, may add artifacts) | 48 kHz PCM (lossless, no cloud processing) | OBS |
| **Artifacts** | Blockiness in dark areas, motion blur during hand gestures | No compression artifacts — clean at 18 Mbps | OBS |
| **Latency Impact** | Near-real-time streaming reduces quality for low-latency delivery | Zero latency — records direct to disk at full quality | OBS |
| **Internet Dependency** | Required for entire recording duration | Only needed for file upload after recording | OBS |
| **Reliability** | Drops connection = lost footage (auto-reconnect, but gap exists) | No internet = no problem | OBS |
| **File Handling** | Auto-uploaded to cloud, auto-sync | Manual file transfer by each host | Riverside |
| **Sync** | Automatic server-side multi-track sync | Manual clap alignment | Riverside |
| **Ease of Setup** | Install, click "Record" | Install, configure settings, add sources, test | Riverside |
| **Host Technical Skill** | Minimal — almost anyone can do it | Medium — needs 5 min of configuration | Riverside |

**THE TRADEOFF EXPLAINED:**

Riverside.fm's magic is **zero-effort multi-track sync**. Each host records locally (yes, Riverside also does local recording), but the cloud layer handles uploading, syncing, and multi-track timeline generation automatically. The tradeoff is that the locally recorded file on Riverside is still compressed to ~8 Mbps for bandwidth efficiency.

OBS local recording gives you **more data per frame** — 18,000 Kbps vs Riverside's ~8,000 Kbps. That difference is visible in:
- Dark areas (gradients instead of blocky compression)
- Facial details (skin texture, eyes)
- Text (on-screen graphics, captions overlays in post)
- Motion (cleaner when someone gestures or moves)

**WHEN THE QUALITY DIFFERENCE MATTERS:**
- **Short-form clips (60s vertical):** Critical. A clean source clip = better-looking final short. Compression artifacts get magnified when you crop/resize for vertical.
- **Broadcast overlays:** The overlay (AIMN branding, ticker, captions) looks sharper on a clean source.
- **Multi-cam transitions:** Clean sources mean cleaner dissolves/wipes between angles.
- **Post-production color grading:** More data = more room for color correction without introducing noise.

**WHEN RIVERSIDE.fm IS STILL USEFUL:**
- If a host's internet is unreliable (OBS file fails to upload)
- For quick testing / one-off episodes where setup time matters more than quality
- As a backup recording (record both OBS and Riverside simultaneously for redundancy)

### 1F — Troubleshooting Guide

**1. OBS not detecting webcam**
- **Windows:** Go to Settings → Privacy & Security → Camera → Allow apps to access camera. Also check OBS is in the allowed list.
- **Mac:** System Settings → Privacy & Security → Camera → Toggle OBS on.
- **Generic:** Unplug and replug the webcam. Restart OBS. If still doesn't show, try "Add Sources" → "Video Capture Device" again — your webcam may be listed under a different name.

**2. Audio desync (lip movements don't match audio)**
- **Cause:** Host's mic and webcam have different sample rates (e.g., mic at 44.1 kHz, webcam at 48 kHz).
- **Fix:** In OBS Settings → Audio → Set Global Audio Devices to same sample rate (48 kHz for everything). Also in Advanced → Audio → check "Force audio sample rate" is set to 48000.
- **Post-production fix:** If the desync already happened, use Resolve's "Change Clip Speed" to stretch/shrink audio to match video.

**3. File too large to upload**
- **If over 2 GB (WeTransfer):** Split the file. Recommend that host use Google Drive instead.
- **If over 15 GB (Google Drive):** Host can upload in two parts. Or use MEGA (20 GB free).
- **Compression option:** If file size is a consistent problem, lower bitrate to 12000 Kbps. It's still 50% better than Riverside.
- **Compression method:** Host can use `ffmpeg -i input.mp4 -c:v libx264 -b:v 12M -c:a aac output.mp4` to recompress to 12 Mbps (still better than Riverside). You can send them a simple script.

**4. Corrupted recording**
- **Before recording:** Check available disk space. OBS needs at least 2x the expected file size free.
- **Example:** 30 min at 18 Mbps = ~4 GB. Ensure 8 GB free on the recording drive.
- **If it happens mid-recording:** The file up to the error point is usually recoverable. Use `ffmpeg -i broken.mp4 -c copy recovered.mp4` to attempt remux.
- **External drive:** If host's internal drive is full, OBS can record directly to an external USB drive (must be fast — USB 3.0 SSD recommended).

**5. OBS says "Encoding overloaded"**
- **Cause:** The computer can't keep up with the encoding demands.
- **Fix:** Lower settings. Drop bitrate to 12000 Kbps, change Preset to P4 (Faster) or even P3. Drop resolution to 1280x720 (720p) as last resort.
- **Hardware encoder:** Ensure Hardware (NVENC/AMF/VT) is selected, not Software x264. Hardware encoding is 10-50x more efficient.

**6. File won't play after recording**
- **Cause:** OBS was force-closed (crash, power loss) before the file finished writing the MP4 header.
- **Fix:** Use "Remux Recordings" in OBS (File → Remux Recordings → Select the .mkv file instead of .mp4). Then use `ffmpeg -v quiet -i broken.mkv -c copy fixed.mp4`.
- **Prevention:** In OBS Settings → Advanced → Recording → Select "mkv" as recording format (mkv survives crashes). OBS can auto-remux to mp4 after recording stops.

**7. Host says "OBS is recording my whole screen, not just my webcam"**
- **Fix:** They added "Display Capture" instead of "Video Capture Device" in Sources. Tell them to remove Display Capture and add Video Capture Device (webcam) instead.
